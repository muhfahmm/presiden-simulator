URAIAN TEKNIS INFRASTRUKTUR JALUR PERDAGANGAN (EKSISTING)
======================================================

Berdasarkan analisis mendalam terhadap direktori `jalur_perdagangan`, sistem rute yang Anda miliki sudah sangat matang secara arsitektur data. Berikut adalah pemetaan komponennya:

1. STRUKTUR DATA RUTE (FOLDER: rute)
-----------------------------------
A. Rute Internasional (rute/international/):
   - Memiliki `hubs.ts` yang mendefinisikan titik-titik pelabuhan strategis dunia.
   - Memiliki `routes.ts` dengan ribuan koordinat titik untuk jalur global. Ini adalah aset data yang sangat berharga untuk tracking antar benua.

B. Rute Regional (rute/regional/):
   - Berisi `AsianRoutes.ts` dan peta koordinat spesifik negara (Vietnam, dsb).
   - Menggunakan sistem interpolasi (`interpolate_route.js`) untuk memastikan pergerakan antar titik rute terlihat halus dan realistis.

C. Rute Utama Domestik (`mainTradeRoutes.ts`):
   - Sudah terdefinisi jalur-jalur emas Indonesia seperti "Selat Malaka - Laut Jawa" dan "Logistik Selat Makassar".

2. VISUALISASI CANVAS (FOLDER: map)
-----------------------------------
- `TradeMapCanvas.tsx`: Komponen ini adalah "mesin utama" yang menggunakan HTML5 Canvas untuk merender peta.
- Sudah mendukung sistem koordinat Lon/Lat (Longitude/Latitude) yang dikonversi ke posisi X/Y layar. Ini memudahkan integrasi data riil dari file rute.

3. REKOMENDASI IMPLEMENTASI LANJUTAN
------------------------------------
A. Integrasi Histori ke Jalur:
   - Saat transaksi dicatat di `HistoriImport/Ekspor`, kita bisa mengambil data Hub asal dan tujuan.
   - Gunakan data rute dari `AsianRoutes.ts` atau `international/routes.ts` untuk memicu animasi kapal di `TradeMapCanvas`.

B. Mekanisme "Active Shipment":
   - Hubungkan `inbox_updated` atau event perdagangan lainnya untuk membuat "instansi kapal" baru di Canvas yang mengikuti rute yang sudah terdefinisi di folder `rute`.

C. Diferensiasi Jalur:
   - REGIONAL: Digunakan untuk perdagangan jarak pendek (Asean). Kapal lebih kecil, waktu tempuh lebih cepat.
   - INTERNASIONAL: Jalur benua. Kapal kargo besar, waktu tempuh lebih lama, namun kapasitas lebih tinggi.

Dibuat berdasarkan: Analisis direktori jalur_perdagangan/
Tujuan: Sinkronisasi Arsitektur Data Rute dengan Visualisasi Game.

4. RENCANA TAHAPAN PENGERJAAN (STRATEGI EKSEKUSI)
-----------------------------------------------

Tahapan implementasi akan mengikuti alur logistik nyata agar data di histori sinkron dengan visual di peta:

TAHAP 1: IDENTIFIKASI TITIK KOORDINAT (SOURCE & DESTINATION)
- Ekspor: Sistem menetapkan "Source" sebagai pelabuhan utama Indonesia (misal: Jakarta/Tanjung Priok dari `mainTradeRoutes.ts`) dan "Destination" sebagai Hub negara mitra yang dipilih user (misal: Bangkok/Laem Chabang untuk Thailand).
- Impor: Sebaliknya, "Source" adalah pelabuhan negara mitra, dan "Destination" adalah Indonesia.
- Fallback: Jika koordinat pelabuhan spesifik tidak ditemukan di `hubs.ts`, sistem akan menggunakan titik tengah koordinat negara tersebut sebagai cadangan.

TAHAP 2: PEMILIHAN JALUR (ROUTE SELECTION)
- Pencarian Jalur: Sistem akan mengecek file `international/routes.ts` atau `AsianRoutes.ts`.
- Jika ditemukan array koordinat yang menghubungkan kedua titik tersebut, sistem akan memuat rute tersebut sebagai "jalur statis" yang harus diikuti kapal.
- Jika rute tidak terdefinisi secara spesifik, sistem akan melakukan "Linier Interpolation" (garis melengkung sederhana) antara pelabuhan asal dan tujuan.

TAHAP 3: INISIALISASI PENGIRIMAN (SHIPMENT TRIGGER)
- Trigger: Saat user mengklik tombol "Konfirmasi" di `ImporEksekusi` atau `EksporEksekusi`.
- Data Object: Membuat objek `activeShipment` baru yang berisi: id, status (dalam perjalanan), total_waktu (berdasarkan jarak Lon/Lat), dan koordinat rute.
- Event: Mengirimkan signal `new_shipment_started` yang akan ditangkap oleh `TradeMapCanvas`.

TAHAP 4: RENDERING ANIMASI PADA CANVAS
- `TradeMapCanvas.tsx` akan membaca daftar `activeShipments`.
- Menggunakan fungsi `requestAnimationFrame` untuk memperbarui posisi ikon kapal di sepanjang koordinat rute secara halus (Smooth Movement).
- Progress akan dihitung dari (Waktu Sekarang - Waktu Mulai) / Total Durasi.

TAHAP 5: PENYELESAIAN & FEEDBACK
- Saat progress mencapai 100% (Kapal sampai di pelabuhan tujuan), sistem akan:
  1. Menghapus Kapal dari Canvas.
  2. Mengirimkan Notifikasi Inbox: "Kapal pengangkut barang telah tiba!".
  3. Memperbarui status di UI Histori Transaksi.

5. REKOMENDASI TEKNOLOGI (UNTUK REALISTIS & CEPAT)
-------------------------------------------------

Untuk mencapai performa tinggi agar map tetap "enteng" meskipun ada puluhan kapal bergerak, berikut adalah teknologinya:

1. HTML5 Canvas (Mesin Utama): 
   - Mengapa? Karena Canvas jauh lebih cepat daripada DOM/SVG untuk merender objek dalam jumlah banyak. Pilihan `TradeMapCanvas.tsx` sudah sangat tepat untuk rendering real-time yang cepat.

2. TypeScript (Struktur Data):
   - Memberikan keamanan tipe data saat kita melakukan kalkulasi matematis Lon/Lat ke posisi X/Y, sehingga meminimalkan bug "kapal hilang" di peta.

3. GSAP atau Framer Motion (Interpolasi Halus):
   - GSAP sangat direkomendasikan untuk animasi performa tinggi. Kita bisa menggunakannya untuk menghitung posisi kapal di sepanjang kurva rute secara "smooth" dan realistis tanpa membuat CPU panas.

4. Web Workers (Opsi Lanjutan):
   - Jika rute perjalanan sangat kompleks (misal: ribuan titik di `routes.ts`), kalkulasi rute bisa dipindahkan ke Web Worker agar Main Thread UI tidak terganggu (mencegah lag).

5. Optimized JSON Data:
   - Data koordinat di folder `jalur_perdagangan` sebaiknya diparsing secara efisien. Menggunakan `coords.json` yang sudah terstruktur akan membuat "Load Time" rute menjadi instan.

Dibuat berdasarkan: Kebutuhan Performa Tinggi & Visual Realistis
Tujuan: Mengoptimalkan Pengalaman User di Hub Perdagangan Nasional.

6. CATATAN MENGENAI PENGGUNAAN PYTHON
------------------------------------

Meskipun visualisasi utama dilakukan di frontend (browser) menggunakan TypeScript, Python tetap bisa memegang peran krusial sebagai "Mesin di Balik Layar":

1. Pemrosesan Data (Pre-processing):
   - Jika kita memiliki data koordinat mentah dari GPS dunia yang sangat besar, Python (menggunakan library `Pandas` atau `GeoPandas`) adalah alat terbaik untuk membersihkan dan mengonversi data tersebut menjadi file `coords.json` yang siap pakai oleh React.

2. Simulasi Ekonomi Kompleks:
   - Jika rute perdagangan memengaruhi harga pasar secara dinamis (Supply & Demand), logika perhitungan ekonomi yang sangat berat bisa dijalankan di Backend Python (FastAPI/Flask) karena Python sangat unggul dalam kalkulasi data dan statistik.

3. Mengapa Tidak Untuk Visualisasi Langsung?
   - Browser secara asli tidak mengeksekusi Python. Jika posisi kapal dihitung di server Python setiap frame (Tick), akan terjadi "Lag" karena adanya delay pengiriman data lewat internet.
   - Untuk visualisasi yang "Cepat & Realistis", pengerjaan posisi kapal harus tetap dilakukan di sisi Client (TypeScript/Canvas) agar gerakan terlihat halus (60-120 FPS).

Kesimpulan: 
- Python = Otak / Processor Data (Backend/Preprocessing).
- TypeScript = Mata / Visualizer (Frontend/Animation).

Dibuat berdasarkan: Analisis Arsitektur Full-stack.
Tujuan: Membagi beban kerja secara efisien antara Client dan Server.
