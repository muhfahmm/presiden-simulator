ARSITEKTUR FINAL: SISTEM LOGISTIK & RUTE PERDAGANGAN (UNIFIED)
=============================================================

Dokumen ini menggabungkan konsep visualisasi rute perdagangan dengan mesin perhitungan estimasi pengiriman pintar ke dalam satu struktur arsitektur yang solid.

1. FILOSOFI ARSITEKTUR
-----------------------
Sistem ini memisahkan antara DATA (rute koordinat), LOGIKA (perhitungan jarak/waktu), dan VISUAL (canvas rendering) untuk memastikan game tetap ringan (high performance) namun sangat realistis.

2. STRUKTUR DIREKTORI TERPISAH (CLEAN MODULARITY)
---------------------------------------------

Untuk menghindari penumpukan file, sistem dibagi menjadi dua modul utama yang mandiri:

A. MODUL: ESTIMASI_LOGISTIK/ (Fokus pada LOGIKA & DATA EKONOMI)
├── 1_calculator/                # Mesin Hitung Smart (Eksperimen Jarak vs Waktu)
│   ├── shippingCalculator.ts    # Logic utama (Transit, Penalti, Hubungan Diplomatik)
│   └── pathfinder.ts            # Algoritma pencari jalur terpendek
├── 2_storage/                   # State Management
│   ├── shipmentStorage.ts       # Database lokal untuk kapal yang sedang aktif
│   └── shipmentTypes.ts         # Definisi tipe data logistik
└── 3_transit/                   # Database Node Transit
    └── transitFriction.ts       # Data penalti waktu tiap negara/bottleneck

B. MODUL: VISUAL_RUTE_MAP/ (Fokus pada RENDERING & ANIMASI)
├── 1_map_engine/                # Rendering Engine
│   ├── TradeMapCanvas.tsx       # Canvas Renderer (Eksisting)
│   └── ShipmentOverlay.tsx      # Info layer di atas peta
├── 2_database_rute/             # Database Koordinat (Eksisting)
│   ├── international/           # Global Routes
│   ├── regional/                # Asian/Regional Routes
│   └── mainTradeRoutes.ts       # Jalur Domestik
├── 3_components/                # UI Visual
│   ├── ShipIcon.tsx             # Aset kapal & pesawat bergerak
│   └── RouteTooltip.tsx         # Detail muatan yang melayang di peta
└── 4_assets/                    # Media
    └── icons/                   # Ikon-ikon visual strategis

3. KORELASI ANTAR MODUL
-----------------------
- MODUL A (Logistik) bekerja mengirimkan "Data Pergerakan" ke MODUL B (Visual).
- MODUL B hanya bertugas merender data yang diberikan oleh MODUL A tanpa tahu cara perhitungannya (Separation of Concerns).

4. INTEGRASI FITUR PINTAR (SMART FEATURES)
-------------------------------------------
- Jarak Riil: Waktu tempuh dihitung otomatis dari total jarak Lon/Lat di folder `2_rute/`.
- Dynamic Transit: Delay bertambah otomatis setiap kali rute melewati "Node" negara mitra.
- Geopolitical Friction: Perjalanan melambat jika melewati wilayah dengan hubungan diplomasi rendah.
- Visual Sync: Posisi kapal di `1_map` selalu sinkron dengan sisa waktu pengiriman di `3_logic`.

5. ALUR KERJA SISTEM (WORKFLOW)
-------------------------------
1. User konfirmasi transaksi -> `3_logic/shippingCalculator` menghitung estimasi.
2. Data masuk ke `3_logic/shipmentStorage` -> Kapal menjadi status "Active".
3. `1_map/TradeMapCanvas` mendeteksi kapal baru -> Kapal muncul di pelabuhan asal.
4. Kapal bergerak sepanjang jalur `2_rute` berdasarkan progres waktu Game.
5. Kapal sampai -> Notifikasi Inbox dikirim -> Stok barang di gudang diperbarui.
