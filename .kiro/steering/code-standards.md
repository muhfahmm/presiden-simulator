# Code Standards & Guidelines

## Documentation Files

**Do NOT create summary or documentation markdown files** unless explicitly requested by the user. This includes:
- Summary files after completing tasks
- Process documentation
- Recap files
- Status update files

Focus on code changes only. Keep the workspace clean and minimal.

## Code Quality

- Write minimal, focused code that directly solves the problem
- Avoid verbose implementations
- Use semantic rename and smart relocate tools for refactoring
- Check diagnostics after code changes
- Follow existing project patterns and conventions

## Response Style

- Be concise and direct
- Avoid repeating information
- Use actionable information over general explanations
- Don't mention execution logs in responses
- Keep summaries minimal - just state what was accomplished in a few sentences
