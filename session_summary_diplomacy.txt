## SESSION 2: DIPLOMATIC PREREQUISITES & EMBASSY-FIRST LOGIC
## ============================================================

### 5. Embassy-First Validation (Triple-Layer Filter)
Three independent filter layers enforce the embassy requirement:

#### Layer 1: getMessages() Filter (Display Layer)
- Location: inboxStorage.ts, getMessages() method
- Aggressive country extraction from: metadata -> subject regex -> source regex
- Embassy category proposals are EXEMPTED from the check (to allow building new embassies)
- Advanced proposals (trade, pact, alliance, defense) from countries without embassies are filtered out
- Redundancy filter: Embassy proposals from countries already having embassies are also removed

#### Layer 2: addMessage() Filter (Insertion Layer)  
- Location: inboxStorage.ts, addMessage() method
- Same aggressive extraction and embassy check
- Blocks proposals at insertion time so they never enter localStorage
- Also includes redundancy filter for existing embassy partners

#### Layer 3: syncFromServer() Filter (Server Sync Layer)
- Location: inboxStorage.ts, syncFromServer() method
- Applies embassy requirement when Go backend pushes inbox data via SSE
- Prevents server-originated proposals from bypassing client-side rules

### 6. EmbassyList.tsx Fix
- Removed overly restrictive isProposal filter that was hiding all informational messages
- Now displays ALL embassy/diplomacy category messages (informational + proposals), matching other tabs

### 7. Redundancy Filter
- Embassy proposals from countries that ALREADY have active embassies (like Amerika Serikat from initial database) are automatically blocked
- Prevents duplicate "build embassy" requests

### 8. Python AI Brain Scripts (ALL NEW)
Created all missing Python AI scripts that the route.ts tries to execute:

#### global_drift.py
- Simulates daily relationship score changes between ALL nations

#### active_actions/improve_relations.py
- NPCs with embassies gradually improve relations with user

#### npc_to_user/tingkatkan_hubungan.py
- NPCs with embassies + high relations can grant financial aid (2% daily chance)

#### npc_to_user/1_pakta_non_agresi.py [CRITICAL]
- ONLY generates pact proposals from countries with ACTIVE EMBASSY (e=1)
- Must NOT already have a pact (p=1)
- Relation score must be >= 45
- Picks 0-2 random eligible countries per day
- Uses variety of pact templates (10 titles, 5 content variations)

#### npc_to_user/2_aliansi_pertahanan.py [CRITICAL]
- ONLY generates alliance proposals from countries with BOTH embassy (e=1) AND pact (p=1)
- Must NOT already have an alliance (a=1)
- Relation score must be >= 60
- Picks 0-1 random eligible country per day
- Enforces diplomatic progression: Embassy -> Pact -> Alliance

#### npc_to_user/3_perjanjian_dagang.py
- ONLY generates trade proposals from countries with ACTIVE EMBASSY (e=1)
- Must NOT already have a trade deal (t=1)
- Relation score must be >= 50

#### npc_to_user/5_tawaran_kedutaan.py
- ONLY generates embassy proposals from countries WITHOUT existing embassy (e!=1)
- Relation score must be >= 40
- Picks 0-2 from top 10 eligible countries

#### npc_to_npc/1_kedutaan_besar.py
- Simulates AI countries building embassies with each other (0.5% daily chance)
- Generates GLOBAL_NEWS events for the news feed
